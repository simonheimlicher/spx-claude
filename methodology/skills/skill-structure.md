# Skill structure

## 1. Understanding Outcome Engineering context

1a. Systematically ingesting context to prepare for a discussion with the user
1b. Systematically ingesting context to prepare for implementation a request from the user

## 2. Authoring Outcome Engineering artifacts

2a. From scratch based on a conversation with the user or from a user prompt and then asking clarifying questions

## 3. Decomposing Outcome Engineering artifacts

3a. Autonomous decomposition from existing requirements and deicions to lower level

## 4. Reviewing and refactoring Outcome Engineering artifacts

4a. Reviewing and then refactoring (moving stuff around) based on a conversation with the user
4b. Factoring out common aspects of requirements and decisions to higher level requirement or decision artifact out of a conversation with the user
4c. Clarifying/augmenting/aligning/deconflicting based on a conversation with the user

## 5. Authoring tests based on Outcome Engineering context

5a. Invoke the ingestion skill and then the language-specific testing skill and then write or modify the tests while updating the spec artifact with references to test files and to adjust outcome specs based on new insights
